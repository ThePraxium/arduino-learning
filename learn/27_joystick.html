Joystick Module

Push the little thumb-stick and your Arduino instantly knows which way you tilted it — and whether you pressed it down. It's the same controller idea that lives in every game pad. Under that cap are two dials and a button, and you'll read all three at once. This is your first taste of reading direction, not just on/off.

1 joystick module 🔌 A0, A1 & pin 2 🔎 analogRead + digitalRead ⏱ ~15 min
X · Y · SW

What you'll build

A joystick module seated on the breadboard and wired to the Mega with just five wires. As you move the stick, the Arduino reads two numbers (X and Y, each 0–1023) and the button state, and streams them to the serial monitor so you can watch the stick move as numbers.

💡

Resting in the middle, both X and Y read about 512 — dead centre of 0–1023. Tilt one way and the number climbs toward 1023; the other way, it falls toward 0.

What you'll learn

  • 1
    analogRead — reading a range

    Not just HIGH/LOW. analogRead hands back a whole number 0–1023 — a smooth scale of how far.

  • 2
    Two axes at once

    X on A0, Y on A1. Together they pinpoint the stick anywhere on a flat plane.

  • 3
    INPUT_PULLUP for the click

    The built-in pull-up makes the button idle HIGH and read LOW when pressed — no extra resistor needed.

Parts at a glance

Refreshingly short. The joystick module is a single board that already has the two dials and the button built in — you seat it on the breadboard and supply five jumper wires from the kit to connect it.

  • Joystick moduleQty 1 · X, Y & button
  • Jumper wiresQty 5 · F-to-M
  • Breadboard+ Mega 2560 & USB
🔗

The module's pins are usually labelled on the board: GND, +5V, VRx, VRy, and SW. Those five names are all you need to wire it correctly.

Before we wire

It's just two dials and a button

Each axis is a potentiometer

Inside the module sit two potentiometers — the same twist-dials from earlier projects. One turns as you push the stick left–right (that's VRx), the other as you push it up–down (VRy). Each feeds a voltage into an analog pin.

analogRead turns that voltage into a number from 0 to 1023. Centred, the dial sits halfway, so you read about 512. The booklet says X swings roughly 0–500 one way and 497–1023 the other — that gap straddling 512 is the centre.

Pressing the stick is a button

Push the cap straight down and a hidden switch closes — that's SW. We read it with digitalRead like any button.

🎯

We set the pin to INPUT_PULLUP, so it idles HIGH all on its own and snaps to LOW the instant you press. No pull-down resistor, no extra wires — the Arduino supplies the resistor internally.

The main event

Wiring it up, step by step

The diagram is drawn the way it sits on your desk: Mega on the left with the USB & power facing away from you, the breadboard on the right with column 1 at the top-left. The joystick module's five header pins seat into row a, then five jumper wires run from the Mega's pins to those holes. Use Prev / Next and watch each wire appear.

Add now: seat the joystick module's five header pins into the breadboard.
module +5V (red) GND (black) VRx (A0) VRy (A1) SW (pin 2)

The module's five pins are labelled +5V, GND, VRx, VRy and SW. Press them into five holes in the same row band so each pin gets its own column.

  1. Set the module so its pins land at about columns 3–7 on the row a edge.
  2. That gives each pin its own column: +5V, GND, VRx, VRy, SW — left to right.
The blue board with the round thumb-stick rides over the f–j half; its header pins reach across into row a where your jumper wires meet them.

Power first. The module's dials and button all need 5 volts before anything can be read.

  1. Red wire: Mega 5V → the module's +5V pin at column 3, row a.
⚡ The module now has power running through it.

Every circuit needs a return path. Ground completes the loop so the dials can produce a steady voltage.

  1. Black wire: Mega GND → the module's GND pin at column 4, row a.
With +5V and GND in place, the module is alive — now we just need to read its three signals.
  1. Wire from the module's VRx pin at column 5, row a → Mega A0.
A0 is an analog input. It will read the left–right dial as a number from 0 to 1023.
  1. Wire from the module's VRy pin at column 6, row a → Mega A1.
Now both axes report in. A0 = left/right, A1 = up/down — two numbers that fix the stick's position.
  1. Wire from the module's SW pin at column 7, row a → Mega pin 2.
Pin 2 uses INPUT_PULLUP, so it idles HIGH and reads LOW when you press the stick down. No resistor required.
🎉 Hardware done! Module seated, five wires in: +5V→5V, GND→GND, VRx→A0, VRy→A1, SW→2.
✅

One golden rule: match the module's printed labels to the Mega. +5V and GND give power; VRx/VRy go to the analog pins (A0/A1); SW goes to a digital pin (2).

The brains

The code, line by line

It's already saved at sketches\27_Joystick. Three reads, one print: grab X, grab Y, grab the button, then send all three down the wire. Every line that prints is what you'll see scroll past in the serial monitor.

27_Joystick.ino

        

        

          
setup() — arm the button & open the link
pinMode(swPin, INPUT_PULLUP) makes pin 2 idle HIGH so the button reads LOW only when pressed. Serial.begin(9600) opens the link the monitor reads, then it prints a greeting.

          
analogRead() — read each axis
analogRead(xPin) and analogRead(yPin) each return a number 0–1023 — how far the stick is pushed on that axis. The two analog pins are the heart of this project.

          
digitalRead() — read the click
digitalRead(swPin) hands back LOW when pressed (pulled to ground) or HIGH when idle. One line catches the button.

          
Print & pace
The Serial.print calls stitch the three values into one tidy line like X:512  Y:498  SW:up. delay(150) spaces the lines out so the monitor stays readable as you move.

        

      

    


    
    

      
Send it to the board

      
Upload & open the monitor

      
Plug the Mega in. From the project root in PowerShell, upload, then open the serial monitor to watch the stick move as numbers:

      

        
PS>.\scripts\upload.ps1 27_Joystick

        
PS>.\scripts\monitor.ps1

        
In the Claude chat you can run either line by prefixing it with ! . The monitor runs at 9600 baud — press Ctrl+C to stop it.

        

          ✓ What success looks like
          
Upload ends with done uploading, then the monitor prints the greeting and a fresh line of numbers as you move and press:

          
done uploading 27_Joystick
Joystick ready - move and press.
X:512  Y:498  SW:up
X:1023  Y:505  SW:up
X:512  Y:500  SW:PRESSED

When something's off

      
Troubleshooting

      
Almost every “it's not working!” is one of these — and the serial monitor usually shows you which.

      

        
📊 X and Y are stuck at 0 or 1023+
The monitor shows the numbers frozen at an extreme and never near 512. Usually +5V or GND isn't connected, so the dials have no voltage to divide.
Fix: Re-seat the red wire on the module's +5V pin and the black wire on GND. With both in place the centred numbers should sit around 512.

        
🔄 Moving left changes the wrong number+
You push left–right but the Y number changes instead of X (or the values feel flipped). The module is just rotated, or VRx and VRy are swapped.
Fix: Make sure VRx goes to A0 and VRy to A1. If it's only orientation, turn the module so “up” faces away from you — nothing's broken.

        
🔌 Pressing the stick does nothing+
The SW field stays on up no matter how hard you press. The signal wire likely isn't on pin 2, or the SW pin is loose.
Fix: Confirm the wire runs from the module's SW pin to Mega pin 2. The sketch uses INPUT_PULLUP, so a good press reads LOW and prints SW:PRESSED.

        
🧪 The monitor is blank or gibberish+
No text, or random symbols instead of numbers.
Fix: The sketch prints at 9600 baud, so run .\scripts\monitor.ps1 (it defaults to 9600). If another monitor is open, close it — only one program can use the port at a time.

      

    


    
    

      
Level up

      
Challenge & extend

      
Got numbers scrolling? Try one of these — small code tweaks, big payoff. Re-upload with the same command after each change.

      

        
Easy
Name the direction
Instead of raw numbers, print which way you're pushing. Add this just before delay(150):
// turn X into words
if (x < 300)      Serial.println("LEFT");
else if (x > 700) Serial.println("RIGHT");
else              Serial.println("centre");
Now the monitor speaks human: LEFT, RIGHT, or centre.

        
A step further
Steer an LED's brightness
Map one axis (0–1023) onto an LED's brightness (0–255) on a PWM pin like 9:
// at the top: int ledPin = 9;
int bright = map(x, 0, 1023, 0, 255);
analogWrite(ledPin, bright);
Push the stick and the light glows brighter — you've built a dimmer you steer by hand.
