Project 10 · Beginner build

Photoresistor

Wave your hand over a little light sensor and an LED dims. Shine a flashlight on it and the LED brightens. This is your first analog input: instead of a yes/no button, the Arduino reads a whole range — a number from 0 to 1023 that says exactly how bright the room is — and turns it into smooth LED brightness.

LDR + 1 LED 🔌 A0 & pin 9 🔎 analogRead + map ⏱ ~15 min

What you'll build

A photoresistor (also called an LDR — light-dependent resistor) wired as a voltage divider, feeding analog pin A0. The Arduino reads how bright the room is as a number, then drives an LED on pin 9 to match. Cover the sensor and the LED fades; light it up and the LED glows brighter. The serial monitor prints both numbers so you can watch the light become a value.

💡

The whole trick: a sensor that changes its resistance with light, plus one line — analogRead(A0) — that turns that change into a number between 0 and 1023.

What you'll learn

  • 1
    Analog vs digital input

    A button was HIGH or LOW — two choices. A photoresistor gives a whole range, read with analogRead as 0–1023.

  • 2
    The voltage divider

    The LDR and a fixed 10k Ω resistor share 5 volts. Their middle point rises and falls with the light — that's what A0 measures.

  • 3
    map() — rescaling numbers

    Take a 0–1023 reading and stretch it to 0–255, the range analogWrite needs to set LED brightness.

Before we wire

Turning light into a number

A resistor that feels light

A photoresistor is just a resistor whose value changes with light. In bright light it lets current through easily (low resistance); in the dark it chokes it off (high resistance). On its own that's invisible — the Arduino can't measure resistance directly. It can only measure voltage.

So we pair the LDR with a fixed 10k Ω resistor and let them split 5 volts between them. That's a voltage divider.

The middle point is the signal

Picture 5V at the top, GND at the bottom, the two resistors stacked in between. The wire from the middle goes to A0. As the LDR's resistance changes, the voltage at that middle point slides up and down — and analogRead turns it into 0–1023.

🔬

Bright room → higher number. Cover the sensor → lower number. Then map() rescales 0–1023 down to 0–255 so analogWrite can set the LED's brightness to match.

The main event

Wiring it up, step by step

The diagram is drawn the way it sits on your desk: Mega on the left with the USB & power facing away from you, the breadboard on the right with column 1 at the top-left. We build entirely on the a–e half (the side nearest the Mega). Use Prev / Next and watch the board light up exactly what to add.

Add now: the two power wires from the Mega to the breadboard rails.
5V (red) GND (black) A0 signal resistor LDR LED

Give the breadboard power and ground first — the divider and the LED both hang off these two rails.

  1. Red wire: Mega 5V → the breadboard's red (+) rail.
  2. Black wire: Mega GND → the blue (−) rail.
⚡ Both rails are live. The whole left edge is now power + ground.

The LDR has two legs and no polarity — either way round is fine. Its legs just need to land in two different columns.

  1. Push one leg into column 1, row a and the other into column 2, row a.
  2. Column 1 will become the 5V side; column 2 becomes the A0 / divider midpoint.
Tip: spread the legs gently so they reach two separate columns. The squiggly face is the light-sensitive window — keep it pointing up.
  1. Short wire from the red (+) rail to column 1, row b — the LDR's 5V leg.
Now 5V feeds the top of the divider. The other LDR leg (column 2) is the middle point we'll read.
  1. 10k Ω resistor (brown–black–orange–gold): from column 2, row b → the blue (−) rail.
This is the key part. The LDR (5V→col 2) and this 10k (col 2→GND) now share 5 volts. Column 2 is the divider midpoint — its voltage rises and falls with the light.
  1. Signal wire from column 2, row c → Mega A0.
A0 now watches the divider midpoint: a bigger number in bright light, a smaller number when the LDR is shaded.
  1. Signal wire from Mega pin 9 → column 6, row a.
  2. LED: long leg (+) in column 6; short leg (−) in column 7.
  3. 220 Ω resistor: from column 7, row b → the blue (−) rail.
🎉 Hardware done! LDR + 10k divider into A0, and an LED with its 220 Ω on pin 9.
✅

Two golden rules: a component's two legs always sit in different columns · the LED's long leg is + (toward pin 9), the short leg − goes through the 220 Ω to the (−) rail. The LDR has no + or − — either way round works.

The brains

The code, line by line

It's already saved at sketches\12_Photoresistor. Three ideas run the whole thing: read the light with analogRead, rescale it with map, then set the LED with analogWrite. Every line that prints is what you'll see scroll past in the serial monitor.

12_Photoresistor.ino

        

        

          
setup() — one driver, one open mic
pinMode(ledPin, OUTPUT) makes pin 9 a driver. Analog input pins like A0 need no pinMode — they're ready to read. Serial.begin(9600) opens the link the monitor reads.

          
analogRead() — measure the light
int light = analogRead(ldrPin); samples the divider midpoint and hands back a number from 0 (dark) to 1023 (bright). That single number is the whole input.

          
map() — rescale 0–1023 to 0–255
map(light, 0, 1023, 0, 255) stretches the reading into the range analogWrite understands. analogWrite(ledPin, brightness) then sets the LED's glow to match.

          
Print, then pause
The four Serial.print lines build one line like light=742 -> brightness=185. delay(120) waits a beat so the monitor stays readable — about eight readings a second.

When something's off

      
Troubleshooting

      
Almost every “it's broken!” is one of these — and the serial monitor usually tells you which.

      

        
📉 The number barely changes when I shade it+
The monitor shows light= stuck near one value no matter what you do. Usually the divider isn't complete — either the LDR isn't really bridging 5V to column 2, or the 10k isn't reaching GND.
Fix: Confirm 5V→column 1 (LDR top), LDR across columns 1–2, and the 10k from column 2 to the blue (−) rail. The A0 wire must tap column 2, the midpoint.

        
🔌 The number is pinned at 1023 (or 0)+
A0 is reading the rail directly, not the midpoint. 1023 all the time means A0 sees 5V; 0 all the time means it sees GND.
Fix: Make sure the A0 wire is on column 2 (the LDR–10k junction), not on column 1 or the rails. The LDR goes to 5V, the 10k goes to GND, A0 taps between them.

        
⚡ The LED never lights, even with good readings+
The numbers move but the LED stays dark. The LED may be backwards, or both legs share a column.
Fix: Long leg (+) in column 6 (toward pin 9), short leg (−) in column 7 toward the 220 Ω. The two legs must be in different columns, and pin 9 must reach column 6.

        
🧪 The monitor is blank or gibberish+
No text, or random symbols.
Fix: The sketch prints at 9600 baud, so run .\scripts\monitor.ps1 (it defaults to 9600). If you opened another monitor, close it — only one program can use the port.

      

    


    
    

      
Level up

      
Challenge & extend

      
Got it working? Try one of these — small code tweaks, big payoff. Re-upload with the same command after each change.

      

        
Easy
Make it a night-light
Invert it: dark room → LED on, bright room → LED off. Flip the map's output ends:
int brightness = map(light, 0, 1023, 255, 0);
// was 0, 255 -> now bright light dims the LED
Now cover the sensor and the LED glows — a real automatic night-light.

        
A step further
Add a darkness threshold
Instead of fading, snap fully on only when it's truly dark (a light switch with a setpoint):
// after reading light:
if (light < 300) {
  digitalWrite(ledPin, HIGH);   // dark -> on
} else {
  digitalWrite(ledPin, LOW);    // light -> off
}
Tune 300 to your room. Print it so you can pick a good cutoff.
