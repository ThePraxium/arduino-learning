Passive Buzzer

Your Arduino has flashed lights and read a button — now it makes music. A passive buzzer is a tiny speaker with no voice of its own: you feed it a frequency and it sings that pitch. Send a string of frequencies in a row and you get a melody. In this build we walk up a C-major scale — C, D, E, F, G, A, B, C — while the serial monitor names each note as you hear it.

1 passive buzzer 🔌 Pin 8 & GND 🎵 tone() & noTone() ⏱ ~10 min

What you'll build

One little buzzer on a breadboard, wired to pin 8 and ground. The Arduino plays an eight-note scale on a loop — each note for about a third of a second — then pauses and repeats. The serial monitor narrates every note (tone: C4 (262 Hz)) so you can read the song while you hear it.

🎼

The whole trick is one function: tone(pin, frequency). Higher number = higher pitch. A melody is just a list of frequencies played one after another.

What you'll learn

  • 1
    Passive vs active buzzer

    An active buzzer beeps one fixed tone on its own. A passive one is silent until you give it a frequency — so it can play any pitch.

  • 2
    tone() and noTone()

    Start a pitch with tone(pin, hz), stop the sound with noTone(pin). Frequency in hertz is the pitch.

  • 3
    Arrays + a for-loop = a melody

    Store the notes in an array and step through them with a for loop — the pattern behind every tune.

Parts at a glance

Wonderfully short list this time — just the buzzer and two wires. The passive buzzer is the flatter one with an open bottom (no sticker over the hole); the active one is usually sealed. If unsure, the passive buzzer is the quiet one until the Arduino drives it.

Before we wire

How a buzzer makes a pitch

Sound is just fast wiggling

Inside the buzzer is a thin metal disc. When the Arduino flips the pin on and off very fast, the disc snaps back and forth, pushing the air — and that pushing is sound. Flip it 262 times a second and you hear the note C4. Flip it faster (523 times) and you hear a higher C, an octave up.

That “times per second” number is the frequency, measured in hertz (Hz). More hertz = higher pitch. That's the entire idea.

Why “passive” matters

An active buzzer has its own little oscillator — give it power and it beeps one fixed tone, end of story. A passive buzzer has no oscillator, so it stays silent until you supply the wiggling with tone().

🎧

That's the trade: the passive buzzer needs more code, but in return it can play any pitch — which means it can play a melody. tone(8, 262) sings C; tone(8, 330) sings E; noTone(8) hushes it.

The main event

Wiring it up, step by step

The diagram is drawn the way it sits on your desk: Mega on the left with the USB & power facing away from you, the breadboard on the right with column 1 at the top-left. We build entirely on the a–e half (the side nearest the Mega). Only two wires — use Prev / Next and watch the board light up exactly what to add.

Add now: the black GND wire from the Mega to the breadboard's (−) rail.
signal / pin 8 GND (black) buzzer

The buzzer needs a return path to ground, so wire up the (−) rail first — the buzzer's quiet leg will hang off it.

  1. Black wire: Mega GND → the breadboard's blue (−) rail.
⚡ The (−) rail is now true ground for the whole board.

The buzzer has two legs. Like any component, each leg sits in its own column so they don't short together.

  1. Press the buzzer in so one leg lands at column 4 and the other at column 6, both in the a–e half.
  2. Find the + mark (or the longer leg) — that's the signal leg. We'll put it at column 4. The plain leg is the − leg at column 6.
Tip: unlike an LED, a passive buzzer is fairly forgiving — if you can't spot the +, it will still play; the tone just won't care which way around it is. But matching + → pin 8 keeps the wiring tidy.
  1. Signal wire from Mega pin 8 → column 4, row a — the buzzer's (+) leg.
This is the wire that carries the music. Every tone() in the code wiggles this pin, and the buzzer turns that wiggle into sound.
  1. Short wire from column 6, row b (the buzzer's − leg) → the blue (−) rail.
🎉 Hardware done! One buzzer, one signal wire to pin 8, one wire to ground. That's the whole circuit.
✅

Two golden rules: the buzzer's two legs always sit in different columns · the (+) leg (with the + mark / longer leg) goes toward pin 8, the plain − leg returns to the (−) rail.

The brains

The code, line by line

It's already saved at sketches\10_Passive_Buzzer. Three small ideas run the whole song: an array of frequencies, a for-loop that steps through them, and tone() turning each number into a note. Every line that prints is what you'll see scroll past in the serial monitor.

10_Passive_Buzzer.ino

        

        

          
The melody, stored in an array
int melody[] = { 262, 294, ... } is a list of frequencies in hertz — C4 up to C5. The matching names[] array holds the human label for each, just for the serial print.

          
tone() — sing this pitch
tone(buzzerPin, melody[i]) tells the buzzer to vibrate at that frequency until told otherwise. It keeps playing on its own while the rest of the loop runs.

          
for-loop — one note after another
for (int i = 0; i < numNotes; i++) walks the index i from 0 to 7, playing each note for the delay(350) before moving to the next. That's how a list becomes a tune.

          
noTone() — the rest between repeats
After the eight notes, noTone(buzzerPin) stops the sound and we print (silence). A delay(800) holds that quiet beat, then loop() starts the scale again.

        

      

    


    
    

      
Send it to the board

      
Upload & open the monitor

      
Plug the Mega in. From the project root in PowerShell, upload, then open the serial monitor to read the notes as you hear them:

      

        
PS>.\scripts\upload.ps1 10_Passive_Buzzer

        
PS>.\scripts\monitor.ps1

        
In the Claude chat you can run either line by prefixing it with ! . The monitor runs at 9600 baud — press Ctrl+C to stop it.

When something's off

      
Troubleshooting

      
Almost every “no sound!” is one of these — and the serial monitor tells you the code is still running even when the speaker is silent.

      

        
🔇 No sound at all, but the monitor scrolls notes+
The code is fine (you see tone: C4 (262 Hz) scrolling), so it's the wiring or the buzzer. Most often the signal wire isn't actually on pin 8, or a buzzer leg has popped out of its column.
Fix: Re-seat both legs in different columns (4 and 6), confirm the signal wire runs from pin 8 to the (+) leg's column, and the other leg's column reaches the (−) rail.

        
🔈 One steady beep, not a scale+
If it just buzzes one unchanging tone, you almost certainly grabbed the active buzzer (it has its own oscillator and ignores tone()).
Fix: Swap in the passive buzzer — the flatter one with the open hole on the bottom (no sticker sealing it). That's Project 9's part, this is Project 8's.

        
🔌 The scale plays but sounds wrong / crackly+
Notes are there but harsh or cutting out. Usually a leg is only half-seated, so the contact flickers as the disc vibrates.
Fix: Push each buzzer leg fully into its hole until it's snug, and make sure the GND wire is firmly in the (−) rail. Loose ground = noisy sound.

        
🧪 The monitor is blank or gibberish+
No text, or random symbols, even though you might hear the buzzer.
Fix: The sketch prints at 9600 baud, so run .\scripts\monitor.ps1 (it defaults to 9600). If another monitor is open, close it — only one program can use the port at a time.

      

    


    
    

      
Level up

      
Challenge & extend

      
Got the scale playing? Try one of these — small code tweaks, big payoff. Re-upload with the same command after each change.

      

        
Easy
Play it back down
After the scale climbs up, make it climb back down — a full up-and-down run. Add a second loop that walks the index in reverse:
// after the up scale, before noTone():
for (int i = numNotes - 2; i >= 0; i--) {
  tone(buzzerPin, melody[i]);
  delay(350);
}
Now it sounds like a real warm-up exercise.

        
A step further
Play a real tune
Swap the scale for the opening of a song you know. Replace the melody[] values with the notes — here's the start of “Twinkle, Twinkle”:
// C C G G A A G
int melody[] = {
  262, 262, 392, 392,
  440, 440, 392
};
Hint: update numNotes and names[] to match the new length.
