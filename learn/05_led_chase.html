The LED chase

Six LEDs in a column. A single soft glow that flows down the row, then back up, forever — the famous Knight Rider / Larson scanner, also called a flowing-water light. You already know how to blink one LED; this lesson shows you how to command six without writing the same line six times. Meet your new best friends: arrays, for-loops, and a smooth fade.

6 LEDs Pins 2–7 🔢 Arrays + loops ⏱ ~25 min

What you'll build

A column of six LEDs wired to pins 2 through 7. The Arduino fades the glow from one LED to the next — LED 1, 2, 3, 4, 5, 6 — then flows back — 6, 5, 4, 3, 2, 1 — and repeats on its own. As one LED dims the next brightens, so it looks like a soft glowing dot smoothly flowing up and down. And the whole time, the board narrates each move in the serial monitor.

💡

In the traffic light you controlled three pins by naming each one. With six (or sixty!) that gets silly fast. This project's big idea: keep all the pins in a list and let a loop do the repeating for you.

What you'll learn

  • 1
    Arrays — a list of values

    Store all six pin numbers in one place: {2, 3, 4, 5, 6, 7}. One name, many values, reached by position.

  • 2
    for-loops — do it N times

    Walk the list one item at a time instead of copy-pasting the same line six times. Forward and backward.

  • 3
    analogWrite — fade, don't snap

    Brightness 0–255 (PWM) lets one glow flow into the next instead of blinking on and off.

Before we wire

One list, one loop — instead of six of everything

The repeating recipe

Each numbered column on the board is one connected node: holes a–e are joined into one column. So an LED's two legs must sit in different columns, or they short together and nothing lights — same rule you already know, just repeated six times down the board.

The wiring is "blink one LED," copied six times and shifted down. Nothing new to learn in the wiring — the new stuff is all in the code.

One shared (−) rail

The blue (−) rail running down the edge is connected all the way along — a shared "bus." Every LED drops into it through its own 220 Ω resistor, and just one black wire ties that rail back to the Arduino's GND.

🎯

Six LEDs, but still only one wire goes back to GND. That single shared rail does all the work — exactly like the traffic light, scaled up. The code's array is the software version of that one tidy pattern.

The main event

Wiring it up, step by step

The diagram is drawn the way it sits on your desk: Mega on the left with the USB & power facing away from you, the breadboard on the right with column 1 at the top-left, columns counting downward. We build entirely on the a–e half (nearest the Mega), stacking the six LEDs down the board so they read top-to-bottom. Don't let "six LEDs" scare you — it's one tiny recipe repeated six times. Use Prev / Next and watch the board light up exactly what to add.

Add now: the power-return wire from GND to the (−) rail.
signal (1 per LED) GND wire 220Ω resistor LED

Before any LED, give "ground" a home. This single wire is what every light returns through.

  1. Black wire: Mega GND → the breadboard's blue (−) rail.
  2. Any hole on that rail works — the whole blue line is one connected node — but keeping it near the bottom stays out of the LEDs' way.
⚡ The power-return path is set. All six LEDs will share this one wire.

This is the one-LED blink you already know, aimed at pin 2 and the top of the board. Learn it once — the next five are identical, just one column further down.

  1. Signal jumper: Mega pin 2 → column 1, row a.
  2. LED: long leg (+) in column 1; short leg (−) in column 2.
  3. 220 Ω resistor: from column 2, row b → the blue (−) rail.
Why those columns connect: the jumper and the LED's long leg share column 1, so pin 2 feeds the LED. The short leg and the resistor share column 2, so the current flows on to the rail. Two columns, never one twice.
✅ One light down. Steps 2–6 are this exact recipe, one column lower each time.
  1. Signal jumper: pin 3 → column 3, row a.
  2. LED: long leg (+) in column 3; short leg (−) in column 4.
  3. 220 Ω resistor: column 4, row b → the (−) rail.
No new GND wire — the resistor drops into the rail you laid in Step 0. That's the shared ground doing its job.
  1. Signal jumper: pin 4 → column 5, row a.
  2. LED: long leg (+) in column 5; short leg (−) in column 6.
  3. 220 Ω resistor: column 6, row b → the (−) rail.
💚 Halfway! Feel the rhythm yet? Same shape, marching down.
  1. Signal jumper: pin 5 → column 7, row a.
  2. LED: long leg (+) in column 7; short leg (−) in column 8.
  3. 220 Ω resistor: column 8, row b → the (−) rail.
  1. Signal jumper: pin 6 → column 9, row a.
  2. LED: long leg (+) in column 9; short leg (−) in column 10.
  3. 220 Ω resistor: column 10, row b → the (−) rail.
  1. Signal jumper: pin 7 → column 11, row a.
  2. LED: long leg (+) in column 11; short leg (−) in column 12.
  3. 220 Ω resistor: column 12, row b → the (−) rail.
Final check — count them: six LEDs, six resistors (each from a short-leg column to the rail), six signal jumpers (pins 2–7), and ONE black wire from the rail to GND.
🎉 Hardware complete! A full column of six. Now let's give it a brain.
✅

Three golden rules (every LED): each LED's two legs go in different columns · the long leg is + (toward the pin jumper), the short leg − goes to the resistor & rail · one 220 Ω resistor per LED, into the (−) rail.

The repetition, at a glance

Every LED is the same recipe shifted down by two columns. Once you see the pattern, the wiring stops being scary — and it's the exact same pattern the code's array captures.

Per-LED wiring map
LED #Arduino pinSignal jumperLED + (long leg)LED − (short leg)Resistor
LED 1pin 2col 1, acol 1col 2col 2 → rail
LED 2pin 3col 3, acol 3col 4col 4 → rail
LED 3pin 4col 5, acol 5col 6col 6 → rail
LED 4pin 5col 7, acol 7col 8col 8 → rail
LED 5pin 6col 9, acol 9col 10col 10 → rail
LED 6pin 7col 11, acol 11col 12col 12 → rail
Spot the pattern: every LED's long leg jumps column 1 → 3 → 5 → 7 → 9 → 11 (always +2); the − leg is always the very next column; and the resistor drops from that column into the rail. That steady +2 march is the wiring version of looping over a list.

The brains

The code, line by line

This sketch is already saved on your computer at sketches\07_LED_Chase. You don't have to type it — just understand it. The core is two ideas — an array (a list of pins) and for-loops (walk the list) — now joined by analogWrite to fade and Serial to watch it live. Every line that prints is what you'll see scroll past in the serial monitor.

The array: a list of pins

            
int ledPins[] = {2, 3, 4, 5, 6, 7}; makes one name that holds six values. The square brackets [] mean "this is a list." Instead of six separate variables, they all live together, in order.

            

              
ledPins — reached by position (index):

              

                [0]2LED 1
                [1]3LED 2
                [2]4LED 3
                [3]5LED 4
                [4]6LED 5
                [5]7LED 6
              

            

          

          
analogWrite() — brightness, not just on/off
digitalWrite only knows full on or off. analogWrite(pin, value) sets a brightness 0–255 using a trick called PWM — exactly what lets an LED fade. On the Mega, pins 2–13 can do PWM, so pins 2–7 are all fine.

          
crossfade() — hand the glow over
Our helper takes a fromIdx and a toIdx. In a tight loop it raises the new LED 0→255 while lowering the old one 255→0 at the same time. Done quickly, your eye reads it as the light flowing from one LED into the next. It also Serial.prints which LED and pin it's lighting.

          
The loops walk the array
Forward, for (int i = 1; i < numLeds; i++) calls crossfade(i-1, i, "forward ") — marching 0→5. Backward counts down with crossfade(i+1, i, "backward") to flow 5→0. Same array, opposite direction — that's the "bounce," and loop() repeats it forever.

          
fadeMs & steps shape the motion
fadeMs is how long one hand-off takes — smaller = faster. steps is how many tiny brightness increments make up that fade — bigger = smoother. Tweak the two numbers at the top to taste.

        

      

    


    
    

      
Send it to the board

      
Upload & open the monitor

      
Plug the Mega in. From the project root in PowerShell, upload, then open the serial monitor to watch the chase narrate itself:

      

        
PS>.\scripts\upload.ps1 07_LED_Chase

        
PS>.\scripts\monitor.ps1

        
In the Claude chat you can run either line by prefixing it with ! . The monitor runs at 9600 baud — press Ctrl+C to stop it. Only one program can hold the port, so close the monitor before re-uploading.

When something's off

      
Troubleshooting

      
Almost every "it's broken!" moment is one of these — and the serial monitor usually tells you which. None of them mean you did anything wrong.

      

        
🔌 One LED in the column never lights (the others chase fine)+
That one LED is almost certainly backwards, or its resistor isn't reaching the (−) rail. The glow visibly "skips over" that slot — yet the monitor still prints its line, proving the code is fine and it's a wiring issue.
Fix: Flip that LED 180° so the long leg (+) is on the pin-jumper column and the short leg (−) faces the resistor. Then confirm its resistor really drops into the blue (−) rail, not an empty hole.

        
🌧 Nothing lights at all+
You probably forgot the shared ground wire — the single black jumper from Mega GND to the (−) rail. Without it, none of the six LEDs have a path home. (The monitor will still scroll happily, since serial doesn't need that rail.)
Fix: Add Step 0: black jumper from GND to the blue (−) rail. Then confirm every resistor reaches into that same rail.

        
🤔 The chase "skips" a position or runs out of order+
A signal jumper is in the wrong column or on the wrong pin. The sketch expects pins 2→3→4→5→6→7 to be LEDs 1 through 6, top to bottom. Compare what the monitor says (LED 3 (pin 4)) to which lamp actually lit.
Fix: Check the wires against the per-LED table: pin 2→col 1, pin 3→col 3, pin 4→col 5, pin 5→col 7, pin 6→col 9, pin 7→col 11. Each pin jumper must land on its LED's + column.

        
🧪 The monitor is blank, or all LEDs glow together+
A blank or gibberish monitor means a baud/port mismatch. If every LED glows steadily with no chase, the sketch likely didn't upload.
Fix: The sketch prints at 9600 baud, so run .\scripts\monitor.ps1 (it defaults to 9600), and close any other monitor first — only one program can use the port. If there's no chase, re-run .\scripts\upload.ps1 07_LED_Chase.

      

    


    
    

      
Level up

      
Challenge & extend

      
Got it bouncing? Brilliant. Try one of these — small code tweaks, big "I made it do that!" payoff. Re-upload with the same command after each change.

      

        
Easy
Change the speed
The fade is governed by one number. Want a frantic scanner? A lazy drift? Edit the top of the sketch:
// was 350
int fadeMs = 120;  // faster, snappier
int fadeMs = 700;  // slower, dreamy
Smaller = faster. For a silkier fade, also raise steps. One named number rules the whole animation.

        
A step further
Add more LEDs to the list
Wire two more LEDs to pins 8 and 9 (same recipe, +2 columns each), then just extend the array and the count — the loops handle the rest automatically:
int ledPins[] = {2, 3, 4, 5, 6, 7, 8, 9};
int numLeds   = 8;
No new loop code needed — that's exactly why arrays + for-loops are worth learning.
